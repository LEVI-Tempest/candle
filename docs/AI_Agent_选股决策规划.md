# AI Agent 选股决策规划（意图冻结版）

## 1. 最终意图（冻结）

> 搭建一个 AI Agent 系统，调用 `candle`（以及后续卡片化分析能力）作为 CLI Tools，完成选股研究与决策支持；  
> **不执行交易**，只输出交易依据、候选标的、风险说明与消息推送结果。

### 1.1 边界约束
- AI Agent 不下单、不撤单、不触发任何交易执行动作。
- 所有结论必须有数据来源与时间戳，不能编造。
- 输出必须结构化，可审计、可回放、可复盘。

## 2. 总体架构（MVP -> 可扩展）

1. `collector`：行情采集与落库（先日线，后扩展分钟线/新闻）。
2. `signal-engine`：调用 `candle` CLI 计算形态、趋势、风险分。
3. `decision-agent`：汇总多因子，生成候选池与交易依据。
4. `notifier`：消息推送（飞书/Telegram/邮件）。
5. `audit-store`：保存每次输入数据、输出结论、版本号与参数。

## 3. 分阶段规划

### Phase 0：基础标准化（1-2 天）
- 定义统一数据结构：OHLCV、symbol、timeframe、source、as_of。
- 定义统一决策输出 JSON Schema（候选、证据、反证、风险、失效条件、置信度）。
- 固定目录与命令约定（`cmd/collector`、`cmd/signal`、`cmd/agent`）。

### Phase 1：可运行 MVP（3-5 天）
- 用 `cron` 每日定时：
  - 采集行情 -> 落库
  - 运行 `candle` 信号识别
  - 生成 Agent 报告
  - 推送消息
- 输出日报：TopN 候选 + 每只股票的依据与风险。
- 增加失败重试与超时告警。

### Phase 2：质量提升（1-2 周）
- 引入事件风险（公告/新闻）作为否决或降权因子。
- 加入回测校验（历史滚动验证信号有效性）。
- 引入“市场状态识别”（趋势/震荡）做策略切换。
- 建立版本化评估：不同 prompt 与评分参数对比。

### Phase 3：工程化（持续）
- 从 `cron` 升级为任务编排（如 Temporal/Airflow/Argo）。
- 增加缓存、并发控制、限流、幂等与全链路监控。
- 多市场扩展（A 股 -> 港股/美股）与多周期联动。

## 4. 采集与调度策略建议

### 4.1 采集策略（先稳后全）
- 优先目标：日线 OHLCV 稳定采集。
- 数据源策略：主源 + 备源（主源异常时回退）。
- 数据健康检查：
  - 时间连续性
  - 异常值检测（高低开收关系）
  - 去重与补洞

### 4.2 调度策略（先 cron）
- `cron` 足够支撑 MVP。
- 建议时序：
  1. 06:00 采集日线
  2. 06:10 跑 `candle` 信号
  3. 06:15 Agent 生成决策摘要
  4. 06:16 推送消息
- 每次运行落审计日志，便于复盘。

## 5. `candle` 需要做的内容（核心清单）

## 5.1 CLI 产品化
- 提供稳定 CLI 子命令（建议）：
  - `candle collect`（可选，或交给独立 collector）
  - `candle signal`
  - `candle score`
  - `candle report`
- 所有命令支持：
  - `--input` / `--output`
  - `--format json`
  - `--as-of`
  - `--symbol` / `--universe`

## 5.2 输出协议统一
- 统一 JSON 字段：
  - `symbol`
  - `patterns[]`（type/position/strength/risk/time）
  - `trend`
  - `score`
  - `evidence[]`
  - `counter_evidence[]`
  - `invalid_if[]`
  - `as_of`
- 增加 schema 校验，避免 Agent 解析歧义。

## 5.3 识别与评分增强
- 将“布尔识别”升级为“可解释评分”主通道。
- 强制输出每个模式的触发条件明细（阈值命中情况）。
- 引入最小强度阈值与市场状态过滤（趋势/震荡分场景）。

## 5.4 图表与证据一致性
- 当前图表标记逻辑需升级为“逐 pattern 精确定位标注”，与输出 JSON 一致。
- 图表上显示：
  - 触发点
  - 强度
  - 风险标签
  - 失效线（可选）

## 5.5 数据与时间处理
- 明确 K 线顺序（升序/降序）并全链路统一。
- 统一时区与交易日历。
- 增加缺失数据处理策略（跳过/补齐/降级）。

## 5.6 测试与可靠性
- 修复当前测试稳定性问题（examples 包编译冲突、趋势测试阈值问题）。
- 增加回归样例集（固定输入 -> 固定输出快照）。
- 为 CLI 增加契约测试（输入输出 JSON 对齐）。

## 6. 决策 Agent 输出模板（建议）

- 市场状态：`trend / range / risk-off`
- 候选列表（TopN）：
  - 结论
  - 证据
  - 反证
  - 风险
  - 失效条件
  - 置信度
- 拒绝列表（为什么不选）
- 免责声明：仅供研究，不构成投资建议

## 7. 完成标准（Definition of Done）

- 每日可自动跑完一轮（采集 -> 信号 -> 决策 -> 推送）。
- 每条推荐都有可追溯证据与参数快照。
- 任意一天结果可复跑重现（同版本、同输入、同输出）。
- Agent 全程不具备交易执行能力。
