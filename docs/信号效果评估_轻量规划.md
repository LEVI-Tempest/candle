# 信号效果评估（轻量规划）

> 目标：不是做完整回测，而是每天回答一个问题：  
> **这些信号，第二天/第3天/第5天到底有没有优势。**

## 1. 定位

当前 `candle` 已经有：

- 信号输出
- `strong / medium / weak`
- `confirm / neutral / contradict`
- `3/5/10` 根前瞻收益记录

所以现在缺的不是“重型回测框架”，而是一个**轻量评估层**。

这个评估层只做：

1. 读取信号记录
2. 统计后续表现
3. 输出日报/汇总

## 2. 输入

第一版直接使用：

- `data/signal_log.csv`

建议字段：

- `time`
- `symbol`
- `pattern`
- `trend`
- `volume_state`
- `decision_score`
- `decision_level`
- `forward_ret_3`
- `forward_ret_5`
- `forward_ret_10`
- `reason`

后续如果要补第二天效果，建议新增：

- `forward_ret_1`

## 3. 输出

建议做两种输出：

### 3.1 JSON

给后续 agent / 脚本消费。

### 3.2 Markdown

给人工每日复盘。

## 4. 最小统计项

每天先只看这些：

- `signals_count`
- `avg_ret_1d`
- `avg_ret_3d`
- `avg_ret_5d`
- `win_rate_1d`
- `win_rate_3d`
- `win_rate_5d`

## 5. 分层统计

至少做三组：

### 5.1 按信号等级

- `strong`
- `medium`
- `weak`

目的：
- 验证高分信号是否真的更强

### 5.2 按量价状态

- `confirm`
- `neutral`
- `contradict`

目的：
- 验证倍量/量价确认是否真的提升效果

### 5.3 按形态类型

- `Hammer`
- `Bullish Engulfing`
- `Morning Star`
- ...

目的：
- 找出哪些形态有效，哪些形态长期失效

## 6. 胜负口径

第一版保持简单：

- 看涨信号：未来收益 `> 0` 记为赢
- 看跌信号：未来收益 `< 0` 记为赢
- 中性信号：先不计入 win rate，只单独看平均波动/收益

不要第一版就引入复杂交易规则、手续费、滑点、仓位管理。

## 7. 展示形式

Markdown 示例：

```md
# Signal Review - 2026-03-13

## Summary
- Signals: 12
- Avg 1D Return: 1.24%
- Win Rate 1D: 66.7%
- Avg 3D Return: 2.81%

## By Level
| level | count | win_1d | avg_1d |
|---|---:|---:|---:|
| strong | 4 | 75.0% | 2.10% |
| medium | 6 | 66.7% | 1.02% |
| weak | 2 | 0.0% | -0.84% |

## By Volume State
| volume_state | count | win_1d | avg_1d |
|---|---:|---:|---:|
| confirm | 5 | 80.0% | 2.45% |
| neutral | 5 | 60.0% | 0.72% |
| contradict | 2 | 0.0% | -1.30% |
```

## 8. 模块建议

建议新增：

- `pkg/review`
- `cmd/review`

职责拆分：

- `pkg/signal`：生成信号
- `pkg/review`：评估信号效果
- `cmd/review`：生成日报

不要把评估逻辑继续塞进 `pkg/signal`。

## 9. 第一版完成标准

满足以下条件即可：

1. 能读取 `signal_log.csv`
2. 能输出总体 `count / avg_ret / win_rate`
3. 能按 `level`、`volume_state`、`pattern` 分层统计
4. 能输出一份可读的 Markdown 日报

## 10. 现在不做的事

- 不做完整资金曲线回测
- 不做手续费/滑点模型
- 不做持仓管理
- 不做策略组合优化

这些是下一阶段问题，不是当前最小需求。

## 11. 结论

当前项目下一步最合理的不是“引入重型 backtest framework”，而是补一个**轻量信号效果评估层**。

先解决：

- 哪类信号有效
- 倍量确认是否真的提升胜率
- 高分信号是否显著优于低分信号

这比直接上复杂回测框架更适合当前阶段。
