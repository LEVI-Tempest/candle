# 量价关系调研（库 / 开源项目 / 理论依据）

> 调研日期：2026-03-09  
> 范围：仅做信息收集，不做代码实现。  
> 目标：为 Candle 项目后续“量价关系”能力建设提供可追溯候选方案。

## 1. 先给结论（TL;DR）

1. **理论层面**：量价关系有长期研究基础，但结论是“有信息增量，不是稳定套利机器”。
2. **工程层面**：最稳妥路线是以 **TA-Lib 指标体系** 为公共语义基线（AD/ADOSC/OBV/MFI 等），再在你们项目内做可解释评分层。
3. **Go 生态层面**：可优先评估 `cinar/indicator`（功能面广 + 含回测框架）和 `markcheno/go-talib`（Go 版 TA-Lib 语义迁移成本低）。
4. **策略框架层面**：Freqtrade / LEAN / Backtrader 都已验证“OHLCV + 指标 + 回测”这条流水线是成熟路径，量价因子可直接嵌入。

## 2. 理论依据（高优先级证据）

### 2.1 技术形态可算法化，不完全是主观图形

- Lo, Mamaysky, Wang (2000, Journal of Finance / NBER WP 7613) 提出系统化、自动化形态识别方法，并在长期样本上发现部分技术指标有增量信息。  
- 对 Candle 的意义：你们“形态 + 量能确认”的方向有学术支撑，但必须强调样本依赖与验证。

### 2.2 量价相关性是经典问题，方向明确但不代表无条件可交易

- Karpoff (1987, JFQA) 综述指出：成交量与价格变动幅度正相关，在股票市场与价格变动本身也存在关系。  
- 对 Candle 的意义：量能应作为**确认层**，而不是替代价格结构本身。

### 2.3 交易量与动量生命周期相关

- Lee & Swaminathan (2000, Journal of Finance) 发现历史成交量/换手率可解释动量与价值风格的联系，并影响动量持续与反转节奏。  
- 对 Candle 的意义：可把“量能状态”当作形态后续表现分层变量，而不是统一阈值。

### 2.4 微观结构理论提供“量驱动价格冲击”解释

- Kyle (1985, Econometrica) 的连续竞价模型给出信息交易、流动性与价格形成关系，是量价冲击研究基础。  
- 对 Candle 的意义：未来若做更细粒度（逐笔/盘口）量价模型，有理论桥梁，不局限于日线指标。

## 3. 可直接使用的库（按与你们项目相关性）

## 3.1 TA-Lib（核心基线）

- 定位：技术指标事实标准之一，核心 C/C++，BSD 许可。  
- 量价相关：函数列表直接含 `AD`、`ADOSC`、`MFI`、`OBV`。  
- 优点：指标定义稳定、跨语言生态广、工程上可作“统一语义层”。

适配建议：
- 把 TA-Lib 指标公式当做 Candle 的“语义对照标准”，避免自研公式漂移。

## 3.2 Go 库：`markcheno/go-talib`

- 定位：TA-Lib 的纯 Go 移植实现。  
- 优点：无 CGO 依赖时更易集成到纯 Go 服务。  
- 风险：仓库发布节奏相对保守（需你们自行做函数级回归验证）。

适配建议：
- 用它快速补齐量价指标原型；关键指标（OBV/CMF/MFI/VPT）要和 TA-Lib 基准做一致性对拍。

## 3.3 Go 库：`cinar/indicator`

- 定位：Go 技术指标 + 策略 + 回测框架的整合库。  
- 量价覆盖：文档列出 CMF、MFI、OBV、VPT、VWAP 等。  
- 优点：除指标外还提供策略/回测上下文，利于快速验证量价规则。

适配建议：
- 若你们希望“指标 + 回测脚手架”一步到位，可优先做 PoC。

## 3.4 Python 生态（用于离线研究）

- `TA-Lib Python wrapper`：量价函数组清晰（AD/ADOSC/OBV）。
- `bukosabino/ta`：明确提供 `OnBalanceVolumeIndicator` 与 `ChaikinMoneyFlowIndicator`。
- `pandas-ta-classic`：文档显示体量大（含 Volume 分类与大量形态），适合做特征筛选实验。

适配建议：
- 不一定进生产，但非常适合作为离线研究基线与参数扫描工具。

## 4. 已实现项目 / 框架样本（可参考架构）

## 4.1 QuantConnect LEAN

- 特点：开源交易引擎，官方文档显示支持 100+ 指标与蜡烛形态。  
- 量价证据：文档中可见 `On Balance Volume`、`Volume Weighted Average Price`，并有 `Chaikin Money Flow` 的用法页面。  
- 参考价值：完整的“数据接入 -> 指标更新 -> 回测/实盘”工程闭环。

## 4.2 Freqtrade

- 特点：策略是 DataFrame 驱动，原生 OHLCV。  
- 量价证据：官方说明蜡烛含 `open/high/low/close/volume`，在 `populate_indicators()` 中加载指标。  
- 参考价值：对你们构建“可解释规则管线”很接近。

## 4.3 Backtrader

- 特点：老牌回测框架，指标生态成熟。  
- 量价证据：官方 recipe 给出 MFI 示例，直接用 `self.data.volume` 计算；README 提到内置指标、TA-Lib 支持。  
- 参考价值：适合快速验证量价逻辑在不同市场数据上的稳健性。

## 4.4 面向“蜡烛 + 量价统计”的轻量项目样本

- `xomarquez27/Candlestick_Patterns`：项目描述明确是从 OHLCV 检测蜡烛与成交量形态，并统计 3/5/10 日后表现。  
- 参考价值：思路上与 Candle 的“形态后验评估”接近，但工程成熟度明显低于 LEAN/Freqtrade/Backtrader。

## 5. 对 Candle 项目的落地启发（仅调研建议，不改代码）

1. 先统一“量价指标字典”与公式来源（建议对齐 TA-Lib 命名/定义）。
2. 量价因子优先级建议：`OBV`、`CMF`、`MFI`、`VWAP 偏离`、`VPT`。
3. 每个形态信号输出应包含：
   - `pattern_score`
   - `volume_confirmation_score`
   - `contradiction_flags`（如价新高但 OBV 未新高）
   - `final_confidence`
4. 回测报告必须单独展示“无量价过滤 vs 加量价过滤”的差异，避免只看最终收益。

---

## 6. 风险与注意事项

1. 库的“可用”不等于“可交易”：不同数据源、复权方式、成交量定义会导致指标偏移。  
2. 量价指标极易出现参数过拟合：必须做样本外 + 不同市场 regime 检验。  
3. 社区库（尤其非官方维护分支）需要锁版本并建立回归基线。
