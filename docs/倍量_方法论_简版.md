# 倍量方法论（简版）

> 目的：给 `candle` 后续实现“倍量”相关规则提供一个短、可执行的定义。  
> 原则：`倍量` 不是单独买卖信号，而是**价格行为的确认器**。

## 1. 一句话定义

“倍量”可以理解为：**当前成交量显著高于近期常态成交量**，常见做法是用当前量除以均量，若达到 `1.5x ~ 2.0x` 以上，则视为倍量或显著放量。

建议在工程里不要把“倍量”机械限定为“刚好 2 倍”，而应统一定义为：

- `vol_ratio = volume / ma_volume_n`
- 当 `vol_ratio >= threshold` 时，视为倍量确认

默认建议：

- `n = 10`
- `threshold = 1.5`

## 2. 核心逻辑

倍量本质上回答的是一个问题：

**这次价格动作，是否有足够多的资金参与。**

所以它只能和价格位置一起解释，不能脱离价格单独看。

- 价格突破关键位 + 倍量：更可能是真突破
- 价格上涨但不放量：更容易是假突破或衰竭上涨
- 价格下跌且倍量：若处于高位，常是兑现/出货信号；若处于下跌初段，常是趋势强化
- 价格横盘末端出现倍量：说明原有平衡被打破，值得重点观察

## 3. 使用原则

### 3.1 倍量只做“确认”，不做“起点”

不要把倍量本身当成信号起点，而是作为以下场景的加分项：

- 形态成立后的确认
- 突破前高/平台/均线后的确认
- 反转K线（如锤头、吞没）后的确认

### 3.2 必须结合位置

同样是倍量，位置不同，含义不同：

- 低位倍量：更偏“吸筹/启动确认”
- 中位倍量：更偏“趋势延续确认”
- 高位倍量：更偏“分歧放大”，需要防出货

### 3.3 看“倍量后是否持续”

单日倍量不够，后续至少要看：

- 次日是否守住突破位
- 回踩是否缩量
- 后续 3/5/10 根 K 是否继续沿原方向运行

## 4. 实现建议（给代码）

建议先做最小版，不要复杂化：

### 4.1 指标定义

- `vol_ma10 = MA(volume, 10)`
- `vol_ratio = volume / vol_ma10`

### 4.2 状态划分

- `confirm`: `vol_ratio >= 1.5`
- `neutral`: `0.8 < vol_ratio < 1.5`
- `contradict`: `vol_ratio <= 0.8`

### 4.3 与价格配合的最低规则

- 若价格突破关键位且 `confirm` -> 上调信号分
- 若价格创新高但量未放大 -> 降权
- 若反转形态出现且 `confirm` -> 上调信号分
- 若高位长阴且 `confirm` -> 风险分上调

## 5. 失效与风险

- 倍量可能来自消息刺激，未必代表趋势延续
- 小盘股/低流动性标的更容易出现失真倍量
- 不能只看绝对量，必须相对近期均量
- 不同市场、不同时段，阈值不能完全固定

## 6. 对 `candle` 的落地建议

第一版只保留三件事：

1. `VOL / MA10`
2. `confirm / neutral / contradict`
3. 倍量只作为 `PatternEvidence.VolumeFactors` 的一个因子

不要第一版就做“倍量柱战法大全”。先把它当成一个**可解释、可回测、可调参**的确认因子。
