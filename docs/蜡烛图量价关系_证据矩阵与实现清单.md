## 3. 支持与反证并存（必须同时写入设计）

### 3.1 支持点

- 某些形态在特定市场与窗口具备统计可分性（非零信息增量）。
- 量价组合（如放量突破/放量反转）通常比单一K线形态更稳健。

### 3.2 反证点

- 形态规则存在数据挖掘偏差风险（data-snooping）。
- 交易成本、滑点、流动性会显著侵蚀形态策略收益。
- 不同市场、不同年代、不同波动 regime 下效果可能反转。

---

## 4. 对当前项目的实现约束（建议作为开发基线）

1. **形态不是信号终点**
   - 输出字段建议拆分：`pattern_signal`、`context_score`、`volume_score`、`final_score`

2. **量价确认必须可解释**
   - 每个触发都记录“触发理由”：
   - 例如：`VOL > MA5*1.5`、`OBV与价格背离`、`突破窗口缺口且放量`

3. **统一回测口径**
   - 固定持有期（如 2/5/10 日）
   - 固定成交假设（手续费/滑点）
   - 固定样本切分（训练期/验证期/样本外）

4. **输出置信度而非绝对结论**
   - `strong / medium / weak` + 证据条目列表

---

## 5. 可落地的第一批规则（供后续编码）

> 下述规则为“文献导向”的实现候选，不代表已被本项目验证。

- **R1 形态基础分**
  - 各形态按历史稳健性给基准分（可后续回测学习）
- **R2 趋势上下文分**
  - 仅在对应趋势环境下增强：如锤头在线性下跌末端才增权
- **R3 量能确认分**
  - 放量阈值：`VOL / MA10 >= 1.5`（可配置）
  - 缩量阈值：`VOL / MA10 <= 0.8`
- **R4 背离修正**
  - 价创新高而 OBV 不创新高 -> 下调看涨形态分
  - 价创新低而 OBV 不创新低 -> 下调看跌形态分
- **R5 风险闸门**
  - 跳空缺口回补失败/极端波动日触发降权或禁入
